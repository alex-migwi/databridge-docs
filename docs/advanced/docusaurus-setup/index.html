Docusaurus Setup

Set up a documentation site for DataBridge using Docusaurus - includes deployment to GitHub Pages, Vercel, and Netlify

DataBridge CLI - Docusaurus Setup Guide

This guide shows how to create a documentation site using Docusaurus with the tutorial content we‚Äôve generated.

Quick Setup

1. Install Docusaurus

# In your databridge project root
cd docs

# Create Docusaurus site
npx create-docusaurus@latest website classic --typescript

# Navigate to the website
cd website

8. Run Development Server

cd website
npm start

Visit http://localhost:3000

9. Build for Production

npm run build

# Preview production build
npm run serve

Deploy to GitHub Pages

1. Update Configuration

In docusaurus.config.ts, set:

url: 'https://your-username.github.io',
baseUrl: '/databridge/',
organizationName: 'your-username',
projectName: 'databridge',

2. Deploy

# Set Git user info
git config --global user.email "you@example.com"
git config --global user.name "Your Name"

# Deploy
GIT_USER=your-username npm run deploy

Your site will be live at: https://your-username.github.io/databridge/

Deploy to Vercel

1. Install Vercel CLI

npm i -g vercel

2. Deploy

cd website
vercel

Follow the prompts. Your site will be live instantly!

Deploy to Netlify

1. Create netlify.toml

[build]
  base = "docs/website"
  command = "npm run build"
  publish = "build"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

2. Connect Repository

  1. Go to https://netlify.com
  2. Click ‚ÄúAdd new site‚Äù ‚Üí ‚ÄúImport an existing project‚Äù
  3. Connect your GitHub repository
  4. Deploy!
